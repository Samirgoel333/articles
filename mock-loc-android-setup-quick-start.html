Enable Developer options and select Mock Loc

Mock Loc needs Android's mock location setting so your phone knows which app is allowed to provide test GPS coordinates. This is a normal Android permission flow for location testing and does not require root access.

Why Android asks this

Android only lets one selected app provide mock GPS data. Choosing Mock Loc tells the system that you trust it for testing routes, places, and location-based workflows.

1

Clear control

You decide exactly which app can mock your location. Other apps cannot quietly take over this setting.

2

No root required

Mock Loc uses Android's official developer setting — no risky system tools.

3

Easy to turn off

Return to Developer options anytime and choose No app when you want real GPS again.

How to enable the setting

The wording can be slightly different on Samsung, Pixel, OnePlus, Xiaomi, Vivo, Oppo, and other Android phones, but the flow is usually the same.

Step 1

Open About phone

Go to Android Settings, open About phone, then find the Build number field. On some phones it is inside Software information.

Step 2

Turn on Developer options

Tap Build number seven times. Android may ask for your PIN, password, or fingerprint before it unlocks Developer options.

Step 3

Select Mock Loc

Return to Settings, open Developer options, tap Select mock location app, and choose Mock Loc from the list.

Quick final check

Once these items are complete, open Mock Loc, pick a place on the map, and press start. Your device will then use the selected mock GPS location.

Before you start mocking

  • Developer options are enabled on your Android phone.
  • Mock Loc is selected under Select mock location app.
  • You returned to Mock Loc and started the location from the map.
